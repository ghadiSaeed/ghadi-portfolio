POS Transactions Insights

Saudi National Bank (SNB)

A comprehensive Power BI dashboard developed during a Data Engineering internship to analyze over 450,000 Point-of-Sale (POS) transactions.

Portfolio

Overview & Objectives

During an intensive 8-week summer internship at the Data Engineering Department of Saudi National Bank (SNB)—the largest banking entity in the Kingdom of Saudi Arabia—I was tasked with developing an analytical pipeline and dashboard for Point of Sale (POS) transactions.

The primary objective was to consolidate fragmented, raw financial data spanning three months into a unified, queryable dataset. This involved robust ETL processes, data cleansing, and dimensional data modeling to produce an interactive Business Intelligence dashboard capable of tracking KPIs, evaluating card network performance, and identifying geographic spending trends.

Dashboard Features & Insights

The final dashboard successfully processed 450K transactions representing 179.88M SAR in volume. Key features included:

  • KPI Tracking: Instant visibility into Total Transaction Count and Total Amount.
  • Card Network Analysis: Donut charts comparing Mada, Visa, and MasterCard by volume and value. Mada accounted for ~70% of the total transaction amount.
  • Geographic Mapping: Interactive map visuals to analyze transactions across different cities and regions.
  • Business Type Leaderboards: Bar charts identifying the top 20 business categories by frequency (e.g., Fast Food, Supermarkets) and monetary value (e.g., Mobile Shops, Electronics).
  • Customer Insights: Ranking the top 20 customers by transaction count and amount to identify high-value demographics.
  • Dynamic Filtering: Global slicers allowing data segmentation by Time Period, Customer ID (CIF), and Relationship Manager (RM).

Technologies Used

  • Data Warehouse: Google BigQuery
  • Data Processing: Advanced SQL, Microsoft Excel
  • Visualization: Power BI, DAX
  • Data Governance Exposure: Axon, IDQ Analyst, Informatica Developer

My Role

  • Reviewed Business Requirement Documents (BRDs) to understand business needs and translate them into technical solutions.
  • Executed ETL workflows, using SQL to handle null values, remove duplicates, and resolve formatting inconsistencies.
  • Designed an Entity Relationship Diagram (ERD) connecting core tables (customers, accounts, transactions).
  • Architected a Star Schema (Fact and Dimension tables) optimized for BI reporting.
  • Developed and refined the interactive Power BI dashboard for presentation to the department manager.

Challenges & Solutions

Handling Large Data Volumes

Consolidating three months of raw data with over 150,000 rows per month posed performance challenges.

Solution

Leveraged Excel for initial consolidation, and then relied on Google BigQuery's scalable infrastructure to handle the heavy computational load during processing.

SQL Query Optimization

Writing complex joins across massive banking datasets initially resulted in slow execution times.

Solution

Optimized query architecture by selecting only the required columns, optimizing joins, partitioning the data, and refining query structures to significantly reduce execution time.

Complex Data Modeling for Power BI

Establishing accurate cross-filtering without creating ambiguous one-to-many cardinality conflicts.

Solution

Transitioned the data from a transactional state to an analytical state by implementing a strict Star Schema, effectively separating measurable facts from descriptive dimensions.

Project Outcomes & Lessons Learned

The successful delivery of the "POS Transactions Insights" dashboard provided SNB stakeholders with a clear, data-driven narrative of consumer spending behaviors. The project demonstrated my ability to take raw, fragmented banking data and engineer it into a reliable, business-ready dashboard.

Beyond technical development, the internship bridged the gap between academic theory and corporate application. It highlighted the critical importance of rigorous data governance, robust ETL testing, and the art of data storytelling—proving that successful data engineering is about translating complex schemas into compelling, actionable business narratives.
