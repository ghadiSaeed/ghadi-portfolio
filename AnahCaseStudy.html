Anah

Arabic AI-Powered Mental Health Support Platform

GitHub Repository Portfolio

Project Overview

Anah is an Arabic-first, AI-powered web application designed to help users reflect on and better understand their emotional well-being. The platform combines guided journaling, mood tracking, Arabic emotion analysis, personalized recommendations, visual emotion insights, daily task support, and an AI-powered conversational assistant in one culturally adapted experience.

The final system uses a hybrid AI architecture: a MARBERTv2-based classifier analyzes Arabic journal entries, while an OpenAI-powered chatbot generates context-aware supportive responses.

  • Type: Graduation Project (University of Jeddah, 2026)
  • Team Size: 5 students

Problem Statement

The project responds to a gap identified through the team's research: Arabic-speaking users have limited access to digital mental-health tools that combine cultural relevance with intelligent emotional analysis and personalized self-reflection features.

Existing solutions often focus on therapist communication or general wellness content, while offering limited Arabic emotion detection, mood tracking, personalization, or interactive support. Anah was designed as a unified Arabic experience bridging this gap as a self-reflection tool.

Key Features

  • Arabic journaling: Structured writing space with prompts, image support, and streaks.
  • Emotion analysis: Sentence-level processing using a MARBERTv2-based model.
  • Emotion detection: Multiple detected emotions ranked using frequency and confidence scores.
  • Mood selection: Emoji-based alternative to journal input.
  • Emotion Analysis Dashboard: Interactive visualizations of detected emotions, emotion distributions, and historical trends.
  • Personalized recommendations: Personalized well-being recommendations based on detected emotions.
  • Emotion-aware quotes: Motivational quotes based on detected mood.
  • AI-powered chatbot: Context-aware, supportive Arabic responses.
  • Message to Yourself: Reflective messaging with scheduling.
  • Daily task checklist: Task tracking with durations.
  • Cloud data: Firebase-backed accounts and secure storage.
  • Privacy focus: Automatic logouts after 5 minutes of inactivity.

Challenges and Solutions

Limited Arabic emotion detection

The initial system relied on rule-based analysis and emoji selection, which could not reliably represent Arabic dialects or mixed emotional states.

Solution

Integrated a MARBERTv2-based classifier, added sentence-level analysis, and combined frequency with confidence scores to identify primary and secondary emotions.

Repetitive chatbot responses

The original rule-based chatbot produced predefined, repetitive responses and struggled with unexpected input.

Solution

Integrated the OpenAI API using gpt-4o-mini through the Flask backend, adding emotional context, lightweight memory, and response constraints.

Conflicting emotion inputs

Journal text and emoji mood selection could create conflicting analysis states across the application.

Solution

Changed the system to accept one input method for each analysis, with strict synchronization logic across the home, journal, and analysis pages.

Mobile usability issues

Complex dashboards, navigation, forms, and chatbot interactions needed to remain usable on smaller screens.

Solution

Redesigned responsive layouts for home, journal, and analysis; adapted navigation; and added chatbot typing indicators for clearer state feedback.
